# v0.3.0 Fix & Architecture Change Plan

> **Architect**: Claude Opus 4.6 | **Date**: 2026-04-04 | **Branch**: `refactor/architecture_design`

---

## 1. Executive Summary

Phase 2 shipped with **4 critical defects** that render the adversarial planning and HIL features non-functional. The root cause of the most severe defect (BUG-01) is architectural: the adversarial SubGraph is invoked via `ainvoke()` inside a wrapper function, which creates an opaque execution boundary that blocks all event propagation to the SSE stream.

**Key architectural decision**: Flatten the adversarial SubGraph — dissolve its 6 nodes directly into the parent StateGraph. This guarantees event propagation, eliminates the state-bridge complexity, and enables the drafter's LLM tokens to stream to the frontend in real time. TutorState gains ~10 adversarial-specific fields (acceptable trade-off for a graph with a single SubGraph).

**Secondary protocol change**: Introduce a new SSE event type `"text"` emitted by the backend for nodes that produce complete text content without LLM streaming (solves both BUG-02 and BUG-04 in one mechanism).

This plan addresses all 10 bugs, 5 architectural concerns, and decomposes Phase 3 into atomic agent tasks.

## 3. Architectural Decisions

### AC-01: SubGraph Event Propagation Strategy

**Decision**: **Option A — Flatten the SubGraph.**

**Rationale**:
1. **Guaranteed event propagation** — all nodes live in one StateGraph; no SubGraph boundary to worry about. `astream_events()` sees everything.
2. **Simplicity** — Option B (native SubGraph streaming) requires a checkpointer on the SubGraph, careful state bridge design (input/output schema mapping between `TutorState` and `PlanAdversarialState`), and version-specific LangGraph behavior. The risk of subtle state-bridging bugs is high.
3. **Acceptable state pollution** — TutorState gains ~10 fields. These are clearly adversarial-planning-specific and don't conflict with existing fields. The project has a single SubGraph; encapsulation overhead is not justified.
4. **Debuggability** — one flat graph is easier to trace in Jaeger/OTel than nested SubGraph events with different namespaces.
5. **HIL interrupt placement** — `interrupt()` can be placed directly in `plan_output` node without any state-bridge gymnastics.

### AC-02: Plan Visibility After HIL Resume

**Decision**: Introduce a new SSE event type `"text"` emitted by the backend for nodes that produce complete text content without LLM streaming.

**Rationale**:
- No unnecessary LLM call — the plan text is already computed; re-generating it wastes tokens and latency.
- General-purpose mechanism — reusable for any non-streaming node (solves BUG-04 for free).
- Clean protocol extension — `"text"` is semantically distinct from `"token"` (complete content vs. incremental chunk).

**SSE Protocol Extension**:
```json
{"type": "text", "content": "<full text content>", "node": "<node_name>"}
```

**Backend implementation** — `app.py:_stream_graph_events`:

Add within the `on_chain_end` handler, AFTER yielding the `node_event` end payload:

```python
        if event_type in ("on_chain_start", "on_chain_end"):
            node_name = event.get("name")
            meta_node = event.get("metadata", {}).get("langgraph_node")
            if node_name and node_name == meta_node and node_name in GRAPH_NODES:
                if event_type == "on_chain_start":
                    # ... existing start logic ...
                    yield f"data: {payload}\n\n"
                else:
                    # ... existing end logic ...
                    yield f"data: {payload}\n\n"

                    # NEW: Emit "text" for non-streaming nodes
                    if node_name in TEXT_EMIT_NODES:
                        output = event.get("data", {}).get("output")
                        if isinstance(output, dict):
                            for msg in output.get("messages", []):
                                if hasattr(msg, "content") and msg.content:
                                    text_payload = json.dumps(
                                        {"type": "text", "content": msg.content, "node": node_name},
                                        ensure_ascii=False,
                                    )
                                    yield f"data: {text_payload}\n\n"
```

**Frontend implementation** — `frontend/app/page.tsx:processSSEEvent`:

Add a new handler block for `"text"` events. IMPORTANT: use **replacement** semantics (not append):

```typescript
if (data.type === "text") {
  setMessages((prev) =>
    prev.map((msg) =>
      msg.id === asstId
        ? { ...msg, content: data.content }
        : msg
    )
  )
  return
}
```

**Why replacement, not append**: The `"text"` event carries the complete content, unlike incremental `"token"` events. For BUG-02 (resume flow), this replaces the drafter's draft tokens with the confirmed plan. For BUG-04 (handle_unknown), this sets the redirect message on an empty placeholder.
